### Currency Conversion Implementation

**Backend (Edge Function):**
```typescript
// supabase/functions/onboard-brand-v4/index.ts

const CURRENCY_MAP: Record<string, string> = {
  US: "USD", CA: "USD", GB: "GBP", AU: "AUD", SG: "SGD",
  DE: "EUR", FR: "EUR", IT: "EUR", ES: "EUR", NL: "EUR",
  JP: "JPY", KR: "KRW", CN: "CNY", IN: "INR", ID: "IDR",
  // ... (100+ mappings)
};

const EXCHANGE_RATES: Record<string, number> = {
  USD: 1.00,
  EUR: 0.92,
  GBP: 0.79,
  AUD: 1.52,
  SGD: 1.34,
  JPY: 149.50,
  // ... (updated weekly)
};

function calculatePricing(tier: string, country: string, billingCycle: string) {
  const currency = CURRENCY_MAP[country] || "USD";
  const rate = EXCHANGE_RATES[currency] || 1.00;

  const basePrice = SUBSCRIPTION_TIERS[tier][`${billingCycle}_usd`];
  const localPrice = Math.round(basePrice * rate);

  return {
    tier,
    currency,
    monthly_price: localPrice,
    yearly_price: localPrice * 11, // 11 months (1 free)
    yearly_savings: localPrice,
    exchange_rate: rate,
  };
}
```

  }).format(amount);

## TIMEZONE MANAGEMENT

### Server Timezone

**Default:** UTC (Coordinated Universal Time)

All timestamps stored in database use UTC:
```sql
created_at TIMESTAMPTZ DEFAULT NOW()  -- Always UTC
```

### User Timezone Detection

**Frontend (JavaScript):**
```javascript
// Automatic timezone detection
const userTimezone = Intl.DateTimeFormat().resolvedOptions().timeZone;
// Example: "America/New_York", "Asia/Tokyo", "Europe/London"

// Store in user profile
await supabase.from('user_profiles').update({
  timezone: userTimezone
});
```

**Backend (Edge Function):**
```typescript
// Convert UTC to user timezone
function convertToUserTimezone(utcDate: string, timezone: string): string {
  return new Date(utcDate).toLocaleString('en-US', {
    timeZone: timezone,
    year: 'numeric',
    month: '2-digit',
    day: '2-digit',
    hour: '2-digit',
    minute: '2-digit',
    hour12: false,
  });
}

// Example:
const utcDate = '2026-02-14T10:00:00Z';
convertToUserTimezone(utcDate, 'America/New_York');  // 2/14/2026, 05:00
convertToUserTimezone(utcDate, 'Asia/Tokyo');        // 2/14/2026, 19:00
convertToUserTimezone(utcDate, 'Europe/London');     // 2/14/2026, 10:00
```

### Daylight Saving Time (DST)

**Automatic Handling:** JavaScript `Intl.DateTimeFormat` and PostgreSQL `TIMESTAMPTZ` automatically handle DST transitions.

**Example:**
```javascript
// March 10, 2026: DST starts in US (UTC-5 â†’ UTC-4)
const beforeDST = new Date('2026-03-09T10:00:00Z');
const afterDST = new Date('2026-03-11T10:00:00Z');

beforeDST.toLocaleString('en-US', { timeZone: 'America/New_York' });
// Output: 3/9/2026, 5:00:00 AM (UTC-5)

afterDST.toLocaleString('en-US', { timeZone: 'America/New_York' });
// Output: 3/11/2026, 6:00:00 AM (UTC-4)
```
